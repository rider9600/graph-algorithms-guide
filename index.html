Graph Fundamentals
What is a graph? Types, terminology, and core concepts.
Beginner

A graph G = (V, E) is a non-linear data structure consisting of a finite set of vertices (nodes) V and a set of edges E connecting pairs of vertices. Graphs model real-world relationships: networks, maps, dependencies, social connections, and more.

⚡ Quick Revision Notes — Fundamentals
Graph = G(V, E) where V = vertices, E = edges connecting them
Directed: edges have direction (A→B ≠ B→A). Undirected: bidirectional.
Weighted: edges have cost/weight. Needed for shortest path algos.
DAG: Directed Acyclic Graph — key for topological sort, scheduling.
Degree of vertex = number of edges. In directed: in-degree + out-degree.
Component: maximal connected subgraph. Use DFS/BFS to count.

Types of Graphs

➡️
Directed Graph
Edges have direction. A→B doesn't imply B→A. Used in dependency graphs, web links.
↔️
Undirected Graph
Edges are bidirectional. If A-B exists, both directions are implied. Friend networks.
⚖️
Weighted Graph
Edges carry numeric weights (cost, distance). Essential for Dijkstra, Bellman-Ford.
🔗
Unweighted Graph
All edges equal. BFS gives shortest path in terms of hop count.
🔄
Cyclic Graph
Contains at least one cycle. Requires cycle detection before topological sort.
🌲
Acyclic Graph
No cycles. Trees are acyclic undirected graphs. DAGs are acyclic directed.
🏗️
DAG
Directed Acyclic Graph. Used for task scheduling, build systems, topological sort.
🔌
Connected / Disconnected
Connected: path between all vertices. Disconnected: isolated components exist.

Core Terminology

Vertex (Node)
Basic unit of a graph. Represents an entity.
Edge
Connection between two vertices. Can be directed or undirected.
Degree
Number of edges connected to a vertex.
In-degree
Directed graphs only: number of incoming edges.
Out-degree
Directed graphs only: number of outgoing edges.
Path
Sequence of vertices where each consecutive pair is connected by an edge.
Cycle
A path that starts and ends at the same vertex.
Component
Maximal connected subgraph within a larger disconnected graph.
Adjacent
Two vertices connected directly by an edge.
Neighbor
All vertices directly connected to a given vertex.
Bridge
An edge whose removal disconnects the graph.
Articulation Point
A vertex whose removal increases number of connected components.

How to store graphs in memory for algorithms.
Beginner
⚡ Quick Revision Notes
Adj Matrix: O(V²) space. Fast edge check O(1). Best for dense graphs.
Adj List: O(V+E) space. Most common. Best for sparse graphs and traversal.
Edge List: Simple. O(E) space. Best for Kruskal's MST algorithm.
Interview default: Always use adjacency list unless told otherwise.
Dense graph: E ≈ V². Sparse graph: E ≈ V.

BFS & DFS
The two fundamental graph traversal algorithms.
Beginner
🔵 Depth-First Search (DFS)
O(V+E)
▼

DFS explores as far as possible along each branch before backtracking. Uses a stack (or recursion). Ideal for cycle detection, path finding, topological sorting, and SCC.

DFS Visualization
Pseudocode
Implementation
Iterative
Pseudocode
DFS(graph, start):
    visited = set()
    
    dfs_helper(node):
        visited.add(node)
        process(node)           # Do work here
        
        for neighbor in graph[node]:
            if neighbor not in visited:
                dfs_helper(neighbor)
    
    dfs_helper(start)
Pseudocode Recursive DFS
function dfs(graph, node, visited=None):
    if visited is None:
        visited = set()
    
    visited.add(node)
    print(node)  # Process node
    
    for neighbor in graph.get(node, []):
        if neighbor not in visited:
            dfs(graph, neighbor, visited)
    
    return visited

# Example usage
graph = {
    0: [1, 2],
    1: [3, 4],
    2: [5],
    3: [], 4: [], 5: []
}
dfs(graph, 0)  # Output: 0 1 3 4 2 5
Pseudocode Iterative DFS (Stack)
function dfs_iterative(graph, start):
    visited = set()
    stack = [start]
    
    while stack:
        node = stack.pop()
        
        if node not in visited:
            visited.add(node)
            print(node)  # Process
            
            # Push neighbors (reverse for same order as recursive)
            for neighbor in reversed(graph.get(node, [])):
                if neighbor not in visited:
                    stack.push(neighbor)
    
    return visited
TimeO(V + E)
SpaceO(V) recursion stack
⚡ DFS Quick Notes
Data structure: Stack (explicit) or call stack (recursive)
Order: Goes deep first, backtracks when no unvisited neighbors
Common mistake: Forgetting to mark visited BEFORE recursing (infinite loop)
Interview trick: DFS finds ALL paths; BFS finds SHORTEST path (unweighted)
Use for: Cycle detection, topological sort, connected components, SCC, maze solving

🟢 Breadth-First Search (BFS)
O(V+E)
▼

BFS explores all neighbors at the current level before moving to the next. Uses a queue. Guaranteed to find the shortest path in unweighted graphs. Processes nodes level by level.

BFS Visualization
Pseudocode BFS
function bfs(graph, start):
    visited = {start}
    queue = deque([start])
    order = []
    
    while queue:
        node = queue.dequeue()
        order.push(node)
        
        for neighbor in graph.get(node, []):
            if neighbor not in visited:
                visited.add(neighbor)
                queue.push(neighbor)
    
    return order

# BFS Shortest Path (unweighted)
function bfs_shortest_path(graph, start, target):
    visited = {start}
    queue = deque([(start, [start])])   # (node, path)
    
    while queue:
        node, path = queue.dequeue()
        if node == target:
            return path
        
        for neighbor in graph.get(node, []):
            if neighbor not in visited:
                visited.add(neighbor)
                queue.push((neighbor, path + [neighbor]))
    
    return []  # No path found
TimeO(V + E)
SpaceO(V) queue
✅
Key insight: BFS visits nodes layer by layer. The first time BFS reaches a node in an unweighted graph, that path is guaranteed to be the shortest.
⚡ BFS Quick Notes
Data structure: Queue (FIFO) — always use deque in Python for O(1) popleft
Shortest path: Only for UNWEIGHTED graphs. For weighted, use Dijkstra.
Level order: Track levels using queue size or sentinel values
Common mistake: Using a list instead of deque (O(n) pop from front)
Multi-source BFS: Add all sources to queue initially for problems like "distance to nearest gate"

⚡ Dijkstra's Algorithm
Intermediate
O((V+E)logV)
▼

Intuition: Greedily pick the unvisited vertex with minimum distance, relax all its neighbors. Uses a min-heap (priority queue). Only works with non-negative weights.

1
Initialize dist[source]=0, all others=∞. Add (0, source) to min-heap.
2
Pop minimum (dist, node) from heap. If already visited, skip.
3
Relax: For each neighbor v, if dist[node] + weight(node,v) < dist[v], update dist[v] and push to heap.
4
Repeat until heap is empty. dist[] contains shortest distances from source.
Pseudocode Dijkstra
function dijkstra(graph, source, V):
    # graph[u] = [(v, weight), ...]
    dist = [float('inf')] * V
    dist[source] = 0
    heap = [(0, source)]   # (distance, node)
    
    while heap:
        d, u = heapq.heappop(heap)
        
        if d > dist[u]:         # Stale entry, skip
            continue
        
        for v, w in graph[u]:
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                heapq.heappush(heap, (dist[v], v))
    
    return dist

# Example: 5-node weighted graph
graph = {0:[(1,4),(2,1)], 1:[(3,1)], 2:[(1,2),(3,5)], 3:[], 4:[]}
print(dijkstra(graph, 0, 5))  # [0, 3, 1, 4, inf]
Time (Binary Heap)O((V+E) log V)
Time (Fibonacci Heap)O(E + V log V)
SpaceO(V + E)
✅ Use Dijkstra When
Single-source shortest path
All edge weights are non-negative
Dense or sparse graphs
Real-world routing (GPS, networks)
❌ Don't Use When
Graph has negative edge weights
Graph has negative weight cycles
All-pairs shortest path needed
Graph is a DAG (use topo sort — faster)
⚡ Dijkstra Quick Notes
Key property: Greedy algorithm. Once a node is popped from heap with distance d, d is finalized.
Why no negatives: Greedy assumption breaks — could miss shorter path via negative edge.
Lazy deletion: Instead of updating heap, push new entry and skip stale ones (check d > dist[u]).
Interview trick: Modified Dijkstra handles problems like "minimum cost to reach target with at most k stops".
🔴 Bellman-Ford Algorithm
Intermediate
O(VE)
▼

Intuition: Relax ALL edges V-1 times. After V-1 iterations, shortest paths are found (since any shortest path has at most V-1 edges). Can detect negative cycles by checking one more iteration.

ℹ️
Why V-1 iterations? A shortest path in a graph with V vertices has at most V-1 edges (no cycles). Each iteration guarantees at least one more edge in the path is correctly relaxed.
Pseudocode Bellman-Ford
function bellman_ford(V, edges, source):
    # edges = [(u, v, weight), ...]
    dist = [float('inf')] * V
    dist[source] = 0
    
    # Relax all edges V-1 times
    for _ in range(V - 1):
        updated = False
        for u, v, w in edges:
            if dist[u] != float('inf') and dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                updated = True
        if not updated:    # Early termination optimization
            break
    
    # Detect negative cycles (V-th iteration)
    for u, v, w in edges:
        if dist[u] != float('inf') and dist[u] + w < dist[v]:
            return None   # Negative cycle detected!
    
    return dist
TimeO(V × E)
SpaceO(V)
⚠️
Negative Cycle Detection: If any distance can still be reduced after V-1 iterations, a negative cycle exists (infinitely reduces path cost).
⚡ Bellman-Ford Quick Notes
Handles: Negative weights, negative cycle detection
Slower than Dijkstra: O(VE) vs O((V+E)logV) — only use when negatives are present
Dynamic programming: dist[i][v] = shortest path using at most i edges (basis of approach)
SPFA: Queue-based optimization of Bellman-Ford, faster in practice but same worst case
🌐 Floyd-Warshall Algorithm
Intermediate
O(V³)
▼

Intuition: Dynamic programming approach. For each pair (i,j), check if going through vertex k gives a shorter path. Solves ALL-PAIRS shortest paths.

Pseudocode Floyd-Warshall
function floyd_warshall(V, adj_matrix):
    # dist[i][j] = shortest path from i to j
    dist = [row[:] for row in adj_matrix]
    
    # Try each vertex k as intermediate
    for k in range(V):
        for i in range(V):
            for j in range(V):
                if dist[i][k] + dist[k][j] < dist[i][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]
    
    # Detect negative cycles: dist[i][i] < 0
    for i in range(V):
        if dist[i][i] < 0:
            return None   # Negative cycle!
    
    return dist
TimeO(V³)
SpaceO(V²)

Topological Sorting
Linear ordering of vertices in a DAG. Two approaches: DFS and Kahn's.
Intermediate

Topological sort produces a linear ordering of vertices in a Directed Acyclic Graph (DAG) such that for every directed edge u→v, vertex u comes before v. Only possible if the graph has no cycles.

Real-world applications: Task scheduling, build systems (Make/CMake), course prerequisites, package dependency resolution, spreadsheet formula evaluation.

⚡ Topological Sort Quick Notes
Only for DAGs: If cycle exists, topological sort is impossible — Kahn's detects this if result size < V.
Not unique: Multiple valid topological orderings may exist.
DFS method: Finish time approach — add to stack when DFS returns from a node.
Kahn's algorithm: Process nodes with in-degree 0 first (BFS-based). Preferred in interviews.
Cycle detection: With Kahn's, if result.length < V, graph has a cycle (Course Schedule I).
DFS Method
Kahn's Algorithm

DFS-based topological sort adds a vertex to a result stack when all its descendants have been processed (post-order). Reversing the stack gives topological order.

Pseudocode DFS Topological Sort
function topo_sort_dfs(V, adj):
    visited = [False] * V
    stack = []
    
    function dfs(v):
        visited[v] = True
        for neighbor in adj[v]:
            if not visited[neighbor]:
                dfs(neighbor)
        stack.push(v)    # Add AFTER all neighbors processed
    
    for v in range(V):
        if not visited[v]:
            dfs(v)
    
    return stack[::-1]   # Reverse for correct order

Kahn's Algorithm (BFS-based): Start with all vertices of in-degree 0. Process them, reducing neighbors' in-degrees. When a neighbor's in-degree becomes 0, add to queue.

Pseudocode Kahn's Algorithm (BFS)
function kahn_topo_sort(V, adj):
    in_degree = [0] * V
    for u in range(V):
        for v in adj[u]:
            in_degree[v] += 1
    
    # Start with all nodes having in-degree 0
    queue = deque([v for v in range(V) if in_degree[v] == 0])
    result = []
    
    while queue:
        u = queue.dequeue()
        result.push(u)
        
        for v in adj[u]:
            in_degree[v] -= 1
            if in_degree[v] == 0:
                queue.push(v)
    
    # If result doesn't contain all V vertices → cycle exists!
    if len(result) != V:
        return []  # Cycle detected
    return result
💡
Kahn's for Course Schedule: If the output length equals V, no cycle exists → valid topological order. If length < V, cycle detected → impossible to complete all courses.
Time (Both)O(V + E)
Space (Both)O(V)

Minimum Spanning Tree
Prim's, Kruskal's, and Union-Find / DSU.
Intermediate

A Minimum Spanning Tree (MST) is a subset of edges of a connected weighted undirected graph that connects all vertices with minimum total edge weight, without forming any cycle. |Edges in MST| = V - 1.

⚡ MST Quick Notes
MST always has V-1 edges for V vertices.
Prim's: Grows tree from source, picks minimum weight edge to add unvisited node. Use for dense graphs.
Kruskal's: Sort all edges by weight, add if doesn't form cycle (DSU). Use for sparse graphs.
Both give same MST weight, may differ in edge choice when weights are equal.
DSU trick: Kruskal's relies on Union-Find to detect cycles in O(α(n)) ≈ O(1) amortized.
🌱 Prim's Algorithm
O((V+E)logV)
▼

Intuition: Similar to Dijkstra. Grow the MST greedily — start from any vertex, always add the minimum weight edge that connects the MST to a new unvisited vertex.

Pseudocode Prim's MST
function prim_mst(V, adj):
    # adj[u] = [(v, weight), ...]
    visited = [False] * V
    min_heap = [(0, 0)]   # (weight, vertex), start at 0
    total_cost = 0
    mst_edges = []
    
    while min_heap:
        w, u = heapq.heappop(min_heap)
        if visited[u]:
            continue
        visited[u] = True
        total_cost += w
        
        for v, weight in adj[u]:
            if not visited[v]:
                heapq.heappush(min_heap, (weight, v))
    
    return total_cost
TimeO((V+E) log V)
SpaceO(V + E)
🔗 Kruskal's Algorithm
O(E log E)
▼

Intuition: Sort all edges by weight. Add edges one by one, skipping any that would create a cycle (checked using Union-Find / DSU).

Pseudocode Kruskal's + DSU
structure DSU:
    function Initialize(n):
        parent = list(range(n))
        rank = [0] * n
    
    function find(x):
        if parent[x] != x:
            parent[x] = find(parent[x])  # Path compression
        return parent[x]
    
    function union(x, y):
        px, py = find(x), find(y)
        if px == py: return False  # Already connected → cycle!
        if rank[px] < rank[py]: px, py = py, px
        parent[py] = px
        if rank[px] == rank[py]: rank[px] += 1
        return True

function kruskal_mst(V, edges):
    # edges = [(weight, u, v), ...]
    edges.sort()  # Sort by weight
    dsu = DSU(V)
    total_cost, count = 0, 0
    mst_edges = []
    
    for w, u, v in edges:
        if dsu.union(u, v):
            total_cost += w
            mst_edges.push((u, v, w))
            count += 1
            if count == V - 1: break   # MST complete
    
    return total_cost, mst_edges
TimeO(E log E)
SpaceO(V)

Strongly Connected Components
Kosaraju's and Tarjan's algorithms for finding SCCs.
Advanced

A Strongly Connected Component (SCC) is a maximal set of vertices such that there is a path from each vertex to every other vertex in the set. Only applicable to directed graphs.

⚡ SCC Quick Notes
Kosaraju's: Two DFS passes. 1st on original graph (finish order), 2nd on transposed graph in reverse finish order.
Tarjan's: Single DFS, uses low-link values and a stack. More space-efficient.
Both O(V+E): Time complexity is the same.
Applications: Finding cycles, condensation graph, 2-SAT problem, web crawling.
Interview tip: Kosaraju's is easier to explain; Tarjan's is preferred in competitive programming.
🔄 Kosaraju's Algorithm
O(V+E)
▼
1
Run DFS on the original graph. Push vertices onto a stack in post-order (finish time).
2
Transpose the graph (reverse all edge directions).
3
Run DFS on the transposed graph in the order vertices are popped from the stack. Each DFS call gives one SCC.
Pseudocode Kosaraju's SCC
function kosaraju(V, adj):
    visited = [False] * V
    finish_stack = []
    
    function dfs1(v):
        visited[v] = True
        for u in adj[v]:
            if not visited[u]: dfs1(u)
        finish_stack.push(v)
    
    for v in range(V):
        if not visited[v]: dfs1(v)
    
    # Transpose graph
    trans = [[] for _ in range(V)]
    for u in range(V):
        for v in adj[u]: trans[v].push(u)
    
    visited = [False] * V
    sccs = []
    
    function dfs2(v, comp):
        visited[v] = True
        comp.push(v)
        for u in trans[v]:
            if not visited[u]: dfs2(u, comp)
    
    while finish_stack:
        v = finish_stack.pop()
        if not visited[v]:
            comp = []
            dfs2(v, comp)
            sccs.push(comp)
    
    return sccs
TimeO(V + E)
SpaceO(V)

Union Find (DSU)
Disjoint Set Union — the most powerful connectivity structure.
Intermediate

Union-Find (Disjoint Set Union) is a data structure that tracks a set of elements partitioned into disjoint subsets. Supports two operations: Find (which set does element belong to?) and Union (merge two sets). Used in Kruskal's MST, cycle detection, and network connectivity.

Pseudocode DSU with Path Compression + Union by Rank
structure DSU:
    function Initialize(n):
        parent = list(range(n))
        rank = [0] * n
        size = [1] * n
        components = n
    
    function find(x):
        if parent[x] != x:
            # Path compression: flatten the tree
            parent[x] = find(parent[x])
        return parent[x]
    
    function union(x, y) -> bool:
        px, py = find(x), find(y)
        if px == py: return False   # Same component → cycle
        
        # Union by rank: attach smaller tree under larger
        if rank[px] < rank[py]: px, py = py, px
        parent[py] = px
        size[px] += size[py]
        if rank[px] == rank[py]:
            rank[px] += 1
        components -= 1
        return True
    
    function connected(x, y) -> bool:
        return find(x) == find(y)
    
    function get_size(x) -> int:
        return size[find(x)]
Find/Union (Amortized)O(α(n)) ≈ O(1)
SpaceO(n)
💡
α(n) is the inverse Ackermann function — grows so slowly it's essentially constant for all practical values of n (n < 10^80).
⚡ DSU Quick Notes
Path compression: Makes every node point directly to root. Flattens tree.

Bridges + Articulation Points
Critical structure

Use this when the problem asks which edge or vertex acts like a single point of failure in an undirected graph.

DFS discovery order low[v] > disc[u] => bridge O(V+E)
  • Bridge: removing the edge disconnects the graph.
  • Articulation point: removing the vertex increases the number of connected components.
  • Interview signal: critical connections, weak links, server failures, road closures.
Pseudocode Bridges (Tarjan)
function find_bridges(V, adj):
    disc = [-1] * V
    low = [0] * V
    timer = [0]
    bridges = []

    function dfs(u, parent):
        disc[u] = low[u] = timer[0]
        timer[0] += 1
        for v in adj[u]:
            if disc[v] == -1:
                dfs(v, u)
                low[u] = min(low[u], low[v])
                if low[v] > disc[u]:
                    bridges.push((u, v))
            elif v != parent:
                low[u] = min(low[u], disc[v])

Bipartite Graph Check
Fast filter

A graph is bipartite if you can color every vertex with two colors so that no edge joins equal colors. No odd cycle means yes.

BFS or DFS works Odd cycle => not bipartite Matching problems
  • Best use: teams, partitions, scheduling, and conflict graphs.
  • Core idea: assign opposite colors to neighbors while traversing each component.
  • Shortcut: every tree is automatically bipartite.
Pseudocode Bipartite Check
function is_bipartite(V, adj):
    color = [-1] * V
    for start in range(V):
        if color[start] != -1:
            continue
        color[start] = 0
        queue = deque([start])
        while queue:
            u = queue.dequeue()
            for v in adj[u]:
                if color[v] == -1:
                    color[v] = 1 - color[u]
                elif color[v] == color[u]:
                    return False
Directed vs undirected
Cycle Detection

The main decision is not the traversal. It is choosing the right state model for the graph type.

Undirected
Track the parent in DFS or BFS. A visited neighbor that is not the parent means a cycle.
Directed
Use three states: unvisited, in recursion stack, finished. A back edge means a cycle.
Common mistake: parent-based logic works for undirected graphs, not directed ones.
Edges vs vertices
Euler vs Hamiltonian

These look similar at first glance, but one has clean degree rules and the other usually drops into exponential search.

Euler path / circuit
Visit every edge exactly once. Degree conditions tell you if it exists. Hierholzer runs in O(V+E).
Hamiltonian path
Visit every vertex exactly once. Usually backtracking or bitmask DP for small inputs.

🌉 Bridges & Articulation Points
▼

A bridge is an edge whose removal disconnects the graph. An articulation point is a vertex whose removal increases the number of connected components.

Found using Tarjan's bridge/AP algorithm with DFS discovery time and low-link values.

Pseudocode Bridges (Tarjan)
function find_bridges(V, adj):
    disc = [-1] * V
    low = [0] * V
    timer = [0]
    bridges = []
    
    function dfs(u, parent):
        disc[u] = low[u] = timer[0]
        timer[0] += 1
        for v in adj[u]:
            if disc[v] == -1:
                dfs(v, u)
                low[u] = min(low[u], low[v])
                if low[v] > disc[u]:
                    bridges.push((u, v))
            elif v != parent:
                low[u] = min(low[u], disc[v])
    
    for v in range(V):
        if disc[v] == -1: dfs(v, -1)
    return bridges

🎨 Bipartite Graph Check
▼

A graph is bipartite if vertices can be colored with 2 colors such that no two adjacent vertices share a color. Equivalent to having no odd-length cycles.

Pseudocode Bipartite Check (BFS)
function is_bipartite(V, adj):
    color = [-1] * V
    
    for start in range(V):
        if color[start] != -1: continue
        color[start] = 0
        queue = deque([start])
        
        while queue:
            u = queue.dequeue()
            for v in adj[u]:
                if color[v] == -1:
                    color[v] = 1 - color[u]
                    queue.push(v)
                elif color[v] == color[u]:
                    return False  # Same color → not bipartite
    return True
🔍 Cycle Detection
▼

Detect whether a graph contains a cycle. Technique differs for directed vs undirected graphs.

Pseudocode Cycle Detection (Directed, DFS)
function has_cycle_directed(V, adj):
    # 0=unvisited, 1=in stack, 2=done
    state = [0] * V
    
    function dfs(v):
        state[v] = 1          # Mark in stack
        for u in adj[v]:
            if state[u] == 1: return True  # Back edge!
            if state[u] == 0 and dfs(u): return True
        state[v] = 2          # Mark done
        return False
    
    return any(dfs(v) for v in range(V) if state[v] == 0)
ℹ️
Undirected: Track parent in DFS. If a neighbor is visited and not the parent → cycle. Directed: Use three states (unvisited/in-stack/done). Back edge = cycle.
🛤️ Euler & Hamiltonian Paths
▼

Euler Path / Circuit

Visits every edge exactly once. Circuit returns to start.

Euler Circuit exists: Every vertex has even degree (undirected), or every vertex has in-degree = out-degree (directed)
Euler Path exists: Exactly 0 or 2 vertices have odd degree (undirected)
Algorithm: Hierholzer's O(V+E)

Hamiltonian Path

Visits every vertex exactly once. NP-Complete — no efficient algorithm known.

⚠️
Hamiltonian Path is NP-Complete. In interviews, use backtracking + pruning or DP with bitmask for small inputs.
